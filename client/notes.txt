
CSS RESET
Problem:
Every browser (Chrome, Firefox, Safari, Edge) has its own default CSS
(e.g. body margin, button styles, font styles).

Solution:
Create reset.css to remove browser defaults so every browser starts from
the same baseline.

Common Reset Rules

*,
*::before,
*::after {
    box-sizing: border-box;
}

* {
    margin: 0;
    padding: 0;
}

Reason:
- Removes default spacing.
- Makes width calculations predictable.
- Applies to both real elements and pseudo-elements.

-----------------------------------------------------------

::before and ::after

Problem:
Sometimes we need an extra element only for decoration
(line, circle, underline, badge, background shape, etc.).

Without pseudo-elements:
Need extra HTML.

<div class="card">
    <span></span>
    Content
</div>

With pseudo-elements:
CSS creates an imaginary element.

.card::before {
    content: "";
}

Advantages
- Cleaner HTML
- Decorative elements stay inside CSS
- Widely used for UI effects

Difference

Pseudo Class
Changes the STATE of an existing element.

:hover
:focus
:first-child

Pseudo Element
Creates an imaginary element.

::before
::after
::placeholder
::selection


DESIGN TOKENS

Problem:
Hardcoding colors, spacing and fonts everywhere makes future changes
difficult.

Instead of

color: #2F6B4F;

30 different places...

Store values once.

:root {
    --credit: #2F6B4F;
}

Use

color: var(--credit);

Now changing one variable updates the whole website.


WHY :root ?
Question:
Where should variables live?

Not Hero.
Not Navbar.
Not Projects.

They should be available to every component.

Therefore use

:root

Think of it as the highest/global scope in CSS.

GLOBAL CONTAINER
.container {

    width: min(100% - 2rem, var(--container-width));

    margin-inline: auto;

}

Purpose

Desktop
↓

Never exceed container width.

Mobile
↓

Automatically shrink to fit.

2rem
↓

Provides left and right breathing space.

margin-inline:auto
↓

Centers the container horizontally.

This single class is used throughout the website.


margin-inline: auto-
Problem:
A fixed-width container stays aligned to the left by default.

.container {
    width: 1000px;
}

Solution:
Tell the browser to automatically divide the remaining horizontal space.

.container {
    width: 1000px;
    margin-inline: auto;
}

Equivalent to:

margin-left: auto;
margin-right: auto;

Purpose:
- Horizontally centers an element.
- Modern replacement for margin-left/right.
- Uses logical properties (works with both LTR and RTL languages).


CSS TRANSITION
Problem:
CSS properties change instantly.

Gray
↓

Green

(No animation)

-----------------------------------------------------------

Solution:
Use transition.

transition DOES NOT change any property.

It only tells the browser:

"If this property changes in the future,
animate the change."

Example

a {

    transition: color 150ms ease;

}

a:hover {

    color: green;

}

Flow

Mouse Hover
↓

:hover becomes active
↓

color changes
↓

transition detects the change
↓

browser animates color over 150ms

Important

:hover changes the property.

transition animates HOW the property changes.

CENTERING A FIXED/ABSOLUTE ELEMENT
left: 50%

Moves the LEFT EDGE of the element to 50% of the parent's width.

It DOES NOT center the element.

-----------------------------------------------------------

transform: translateX(-50%)

Moves the element left by 50% of ITS OWN width.

Unlike left:50%, this percentage is calculated using the
element's width, NOT the parent's width.

Combining both:

left: 50%;
transform: translateX(-50%);

Result:

1. Left edge reaches the center.
2. Element moves back by half of its own width.
3. The CENTER of the element is now perfectly aligned with
   the center of the screen.

Remember:

left:50%      → Parent's width
translateX()  → Element's own width

:last-child-
Purpose:
Selects the last child element inside a parent.

Example

<div>

    <span></span>

    <span>Available for work</span>

</div>

span:last-child

↓

Selects only the second span.

Useful when an element can be uniquely identified by its position
instead of adding another class.
